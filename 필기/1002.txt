	컴파일언어: 기계어 번역과정을 거쳐 소스를 실행	ex) Java	*.java=>*.class
		  
	인터프리터언어: 바로 번역 후  실행 ex) JavaScript
		   
원시 프로그램: 사람이 작성한 소스
	>>기계어로 번역(0,1 조합): 컴파일 //ctrl+f11(컴파일 후 실행)
목적 프로그램: 기계가 이해할 수 있는 소스


Java: JDK-원시 프로그램과 관련
       JRE-목적 프로그램과 관련

목적프로그램은 실행중인가? >>No
     실행중인 프로그램은 주기억장치에 상주하는 프로그램
     목적 프로그램은 원시프로그램을 기계어로 번역 후 보조기억장치에 저장
     스레드(CPU)가 목적프로그램을 실행시킴
     목적 프로그램은 스레드 단위로 실행됨

JVM구조에 포함된 영역
Stack: 메서드 Call, FILO구조, 지역변수가 저장
Heap: 객체(인스턴스) 생성과 사용, 소멸에 관여
Method: 소스파일(클래스로드 시), 멤버변수, static 저장
======================================================
-static: 공용변수// 객체를 만들지 않아도 접근가능
		객체를 여러 개 생성해도 하나의 변수를 공유
-final: 초기값이 설정된 이후 변경불가

//싱글톤 공부하기
싱글톤(Singleton) 패턴이란?
싱글톤 패턴은 특정 클래스의 인스턴스를 1개만 생성되는 것을 보장하는 디자인 패턴이다. 
즉, 생성자를 통해서 여러 번 호출이 되더라도 인스턴스를 새로 생성하지 않고 
최초 호출 시에 만들어두었던 인스턴스를 재활용하는 패턴이다.
