--객체

토론주제
1) 쭈꾸미 레시피는 실제 사람이 먹을 음식이라고 할 수 있을까? 
	=No
2) 레시피는 왜 만들었을까?
	=재사용성을 높이기위해서
3) 레시피와 실제 만들어진 음식은 어떤 차이가 있을까?
	=레시피는 설명서, 음식은 실제 필요한 것
레시피>>클래스
레시피는 재료와 조리과정
재료>>멤버변수
조리과정>>메서드
음식>>객체

토론 lv2
1) 레시피는 1개만 있고 레시피를 통해 만들어진 음식도 1개라고 말할 수 있을까?
	=상황에 따라 달라짐. 1개만 만들면 1개, 여러 번 만들면 X
2) 레시피가 있다면 음식도 만들어진 것일까?
	=X
3) 서로 다른 쭈꾸미 레시피로 각각 요리를 했다면 음식은 쭈꾸미라는 성격을 갖는다
이 음식은 서로 같은 음식이라고 할 수 있을까?
	=쭈꾸미라는 관점에서 본다면 같은 음식이라고 할 수 있지만 음식의 관점에서 보면 다른 음식

	>>객체는 서로 독립적

토론 lv3
1) 레시피는 여러 사람과 공유할 수 있다. 
이 때 레시피는 하나이고 레시피를 참조하는 사람도 1명일까?
	=공유는 자원의 낭비를 줄일 수 있다.
		>>객체는 독립적이지만 공유하면 메모리의 사용량을 줄일 수 있다.

2) 여러 사람이 하나의 레시피를 참조할 때, 어떤 사람이 레시피를 수정한다면 
다른 사람은 수정된 레시피를 볼까?아니면 수정되지 않은 상태의 레시피를 볼까?
	=수정된 후, 공유는 변경된 후 내용을 본다. 

결론
레시피: 클래스, 음식: 객체
레시피로 사람이 만들어야 음식이 된다.
클래스는 new 연산자로 객체를 만들어낸다.

=====================================
원시변수는 "실제값"을 저장
참조변수는 내가 원하는 객체(객체는 heap 영역에 저장됨)가 있는 "주소"를 저장
"." >> 참조하는 연산자
변수가 저장한 주소로 객체가 있는 곳 참조
클래스명과 생성자는 이름이 같아야한다.


**암기**
-클래스는 객체를 만들기위해 정의한 것
-클래스는 변수와 메서드로 정의
-클래스로부터 만든 객체는 서로 독립적
-객체를 참조하는 변수는 원시변수가 아닌 참조변수
-참조변수는 주소를 저장, 객체는 자바의 힙영역에 저장
-참조변수가 저장하고 있는 주소값을 참조하여 객체에 접근
-new 연산자로 객체를 만들 때는 생성자로 만든다.
-생성자는 객체를 만들 때 처음 호출되는 메서드이다.
